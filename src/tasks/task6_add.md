GameField – класс, отвечающий за игровое поле (наполнение, отображение)

GameFieldElement – класс, работающий со стандартными игровыми элементами

BonusElement (extends GameFieldElement) – класс, работающий со «спец» элементами (бонусами)

Combinations – класс, отвечающий за обработку результативных комбинаций элементов

UserCommand – класс для команд пользователя

CommandHandler – класс, отвечающий за обработку команд (но вообще с двумя этими классами, если честно, немного не уверен)

GameStatistics – класс работы со статистикой игры (история ходов, баллы)

FieldElementCollection - коллекция игровых элементов для использования в классе GameField

1. PlayingProcess (или GameEngine) - "Машина состояний". Управление процессом игры на основе состояний системы. (предварительно состояния WATING_FOR_USER, REFILLING, CHECKING)
2. GameCommand - реализация паттерна Command - и соответственно конкретные команды (UserTurnCommand, UndoCommand(?), ExitCommand, ReRunGameCommand)
3. EventObserver - реализация паттерна Observer (издатель, я так понимаю, PlayingProcess) - будет использоваться для
    - подсчета очков (наблюдатель ScoreCounter),
    - сохранения истории (наблюдатель HistoryWriter),
    - статистики (наблюдатель GameStatistics).

Отсев по второму признаку (если он отвечает на вопрос "этот класс делает ...", если он "выполняет нечто")

BonusElement - продолжает иерархию (от GameFieldElement)

Отсев по шестому признаку (если не содержит команды, а только запросы)
